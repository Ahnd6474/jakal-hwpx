지원 기능

표·이미지·수식까지 전부 지원

단순 텍스트 치환부터 복잡한 문서 구조 편집까지.

📋

표(Table) 편집

set_cell_text()로 승인된 셀 좌표만 수정. 행 추가, 병합 셀 안전 처리.

🖼️

이미지(Picture) 삽입·조회

pictures()로 삽입된 이미지 메타 확인. 캡션 편집 지원.

🔢

수식(Equation) 편집

LaTeX/OMML 수식 스크립트 직접 편집. equations()로 전체 접근.

🔤

텍스트 치환 · 문단 편집

replace_text()로 플레이스홀더 치환. set_paragraph_text()로 문단 교체.

🔖

북마크 · 하이퍼링크 · 필드

북마크, 하이퍼링크, 계산필드, 상호참조, 메일머지 필드 생성·편집.

📄

페이지 설정 · 머리글/바닥글

여백, 용지 크기, 머리글·바닥글 편집. 섹션별 독립 설정.

✅

4단계 검증 시스템

xml_validation_errors · reference_validation_errors · save_reopen_validation_errors 자동 검증.

📝

각주 · 미주 · 자동번호

notes()로 각주·미주 편집. 자동번호(hp:newNum) 갱신 지원.

🎨

도형(Shape) · 스타일

도형 텍스트 편집, 문단·문자 스타일 일괄 적용.

from jakal_hwpx import HwpxDocument

doc = HwpxDocument.open("졸업논문양식.hwpx")
doc.set_metadata(
    title="고닉 특성 분석",
    creator="수능치는아랴양"
)
doc.replace_text("{{제목}}", "고닉 특성 분석")
doc.set_preview_text("졸업논문 초록 요약...")
doc.save("논문_완성.hwpx", validate=True)
AI 안전 규칙

ChatGPT가 jakal-hwpx를 사용할 때 반드시 지켜야 할 19가지 규칙. 프롬프트에 포함하세요.

  • 🚫HwpxDocument를 범용 편집기처럼 직접 쓰지 말 것
  • 🚫save(validate=False)를 절대 호출하지 말 것
  • 🚫저장 후 검증 없이 파일을 반환하지 말 것
  • ✅validation_errors() · xml_validation_errors() · reference_validation_errors() · save_reopen_validation_errors() 중 하나라도 실패하면 결과 파일을 폐기할 것
  • 🚫save_reopen_validation_errors() 호출 실패나 예외를 무시하지 말 것
  • 🚫add_part() · remove_part() · 내부 XML/XPath 직접 수정 · ZIP 후처리 금지
  • 📌replace_text()는 짧고 고유한 플레이스홀더 치환에만 사용할 것
  • 🚫사람용 지시문 문장 전체를 replace_text()로 바꾸려 하지 말 것
  • 📌문단 교체는 승인된 문단 슬롯에 대해서만 set_paragraph_text() 사용
  • ⚠️set_paragraph_text()는 헤더/도형/복합 컨트롤 문단에 사용하지 말 것
  • 📌표 수정은 승인된 셀 좌표에 대해서만 set_cell_text() 사용
  • 🚫병합 셀 구조를 추측해서 수정하지 말 것
  • 📌새 문단/섹션 추가는 템플릿이 허용한 위치에서만 수행
  • 🚫배포보호 문서(is_distribution_protected)는 수정하지 말 것
  • ✅플레이스홀더 치환 후 잔여 지시문이 남아 있으면 실패로 처리
  • ✅치환/수정 성공 개수 확인, 예상 개수와 다르면 실패로 처리
  • 🚫검증 실패 시 우회 저장 · 강제 저장 · 재시도 저장 금지
  • 📌출력은 항상 새 파일로 저장, 원본 템플릿 덮어쓰기 금지
  • 💡가능하면 set_preview_text()를 채울 것
검증 파이프라인 ● 정상 
# 1. XML 구조 검증
errors = doc.xml_validation_errors()
assert not errors, errors

# 2. 참조 무결성 검증
ref_errors = doc.reference_validation_errors()
assert not ref_errors, ref_errors

# 3. 전체 구조 검증
val_errors = doc.validation_errors()
assert not val_errors, val_errors

# 4. 저장 후 재열기 검증
reopen_errors = doc.save_reopen_validation_errors(
    "output.hwpx"
)
assert not reopen_errors, reopen_errors
