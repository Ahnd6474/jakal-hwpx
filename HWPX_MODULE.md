# jakal_hwpx

Python module for reading, editing, validating, and writing HWPX packages.

## What it does

- opens existing HWPX zip packages
- exposes typed part objects that mirror package files
- preserves untouched parts as original bytes for safer round-trip writes
- supports text, paragraph/style, section/page, table, image, header/footer, note, bookmark, field, and shape edits
- validates package consistency before writing
- round-trips the local HWPX corpus used in this repository

## Package layout

- `HwpxDocument`: package-level document object
- `ContentHpfPart`: `Contents/content.hpf`
- `HeaderPart`: `Contents/header.xml`
- `SectionPart`: `Contents/sectionN.xml`
- `SettingsPart`: `settings.xml`
- `ContainerPart`: `META-INF/container.xml`
- `ManifestPart`: `META-INF/manifest.xml`
- `PreviewTextPart`: `Preview/PrvText.txt`
- `BinaryDataPart`: `BinData/*`
- `HwpxXmlNode`: namespace-aware XML node wrapper for low-level editing

## Example

```python
from pathlib import Path

from jakal_hwpx import HwpxDocument

source = Path("example.hwpx")
target = Path("example-edited.hwpx")

doc = HwpxDocument.open(source)
doc.set_metadata(title="Edited title", creator="jakal_hwpx")
doc.replace_text("old text", "new text", count=1)
doc.append_paragraph("Paragraph added from Python.")
doc.set_preview_text("Preview text generated by Python.")
doc.add_or_replace_binary("unit-test.bin", b"abc123")
doc.save(target)
```

## Feature-level editing

```python
doc = HwpxDocument.open("example.hwpx")

# header / footer
doc.headers()[0].set_text("Edited header")
doc.footers()[0].set_text("Edited footer")

# section / page
section = doc.section_settings(0)
section.set_page_size(width=60000, height=84000)
section.set_margins(left=8500, right=8500)

# paragraph / style
doc.append_paragraph("Paragraph added from Python.")
doc.apply_style_to_paragraph(0, 0, style_id="0", para_pr_id="0", char_pr_id="0")
doc.apply_style_batch(section_index=0, text_contains="target", style_id="0")
doc.styles()[0].set_name("Edited style")
doc.paragraph_styles()[0].set_alignment(horizontal="CENTER")
doc.character_styles()[0].set_text_color("#112233")

# table
table = doc.tables()[0]
table.set_cell_text(0, 0, "Edited cell")
table.append_row()
table.merge_cells(0, 0, 0, 1)

# picture / shape
picture = doc.pictures()[0]
picture.shape_comment = "Edited image comment"
picture.replace_binary(picture.binary_data())
doc.shapes()[0].shape_comment = "Edited shape"

# notes / fields / references
doc.notes()[0].set_text("Edited note")
doc.append_bookmark("python_anchor")
doc.append_hyperlink("https://example.com", display_text="Example")
doc.append_mail_merge_field("customer_name", display_text="CUSTOMER_NAME")
doc.append_calculation_field("40+2", display_text="42")
doc.append_cross_reference("python_anchor", display_text="Anchor Ref")

# equation
doc.equations()[0].script = "x=1+2"

doc.save("example-edited.hwpx")
```

## Protected packages

Some HWPX files are distribution-protected and keep section/header/settings entries encrypted.
`jakal_hwpx` can preserve those files and round-trip them, but high-level editing is disabled for the encrypted parts.

## Validation

Run:

```powershell
python -m pytest -q
```

This repository currently includes:

- unit tests for open/edit/save/reopen flows
- corpus round-trip tests against the valid HWPX files collected in `all_hwpx_flat`
- feature tests for header/footer, notes, bookmarks, fields, equations, shapes, tables, styles, and validation layers
Validation APIs:

- `doc.xml_validation_errors()`: well-formedness and structural checks for core XML parts
- `doc.schema_validation_errors({...})`: optional XSD validation for user-supplied schema files
- `doc.reference_validation_errors()`: manifest/style/field/bookmark reference integrity checks
- `doc.save_reopen_validation_errors()`: save and reopen regression check
