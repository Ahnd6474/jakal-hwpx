# Sample Files

Repository sample documents are grouped by source format:

- `hwpx/`: editable HWPX samples
- `hwp/`: original HWP samples used for conversion tests
- `pdf/`: PDF samples used for bridge validation

Synthetic fixtures generated by repository code live under both `hwpx/` and `hwp/`.
Rebuild the HWPX fixtures with `.\.venv\Scripts\python.exe scripts\augment_sample_corpus.py`.
Rebuild the native HWP control fixtures with `.\.venv\Scripts\python.exe scripts\build_native_control_samples.py`.
The generated corpus currently fills coverage gaps for footer, footnote, endnote, bookmark, hyperlink field scenarios, and native HWP control authoring.

Generated or temporary outputs should go under `build/validation/` or another ignored build directory, not back into the repo root.

Unless a sample file explicitly says otherwise, do not assume that the repository's MIT license relicenses the document contents themselves. Treat sample files as test fixtures or reference material.

Release-blocking sample expectations are fixed in [`../../stability_contract.json`](../../stability_contract.json).
That manifest classifies the corpus by category and records the expected text, controls, and section counts that the release gate enforces.
