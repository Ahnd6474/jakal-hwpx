# Sample Files

Repository sample documents are grouped by source format:

- `hwpx/`: editable HWPX samples
- `hwp/`: original HWP samples used for conversion tests
- `pdf/`: PDF samples used for bridge validation

Synthetic fixtures generated by repository code also live under `hwpx/`.
Rebuild them with `.\.venv\Scripts\python.exe scripts\augment_sample_corpus.py`.
The generated corpus currently fills coverage gaps for footer, footnote, endnote, bookmark, and hyperlink field scenarios.

Generated or temporary outputs should go under `build/validation/` or another ignored build directory, not back into the repo root.

Unless a sample file explicitly says otherwise, do not assume that the repository's MIT license relicenses the document contents themselves. Treat sample files as test fixtures or reference material.
